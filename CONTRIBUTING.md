# Contributing to MOGC

Thank you for your interest in contributing!

What we expect

- Create a GitHub issue describing the bug or feature before starting major work.
- Fork the repository, create a descriptive branch (e.g., `feat/login-page` or `fix/student-route`).
- Follow the existing code style. Run linters and formatters before opening a PR.
- Write concise commit messages and include tests where applicable.

Working locally

- Run the frontend: `cd client && npm install && npm run dev`
- Run the backend: `cd server && pipenv install && pipenv shell && python -m flask run`

Pull request process

- Open a PR against the `main` branch (or the branch described in the issue).
- Describe the change, include screenshots if the UI is affected, and reference related issues.
- Address review comments and keep commits focused and rebased as needed.

Code of Conduct & License

- This repository follows a basic contributor-friendly code of conduct. Please be respectful.
- This repository includes a `LICENSE` file. See `LICENSE` for the license details; contact the maintainers if you need clarification before making large contributions.
